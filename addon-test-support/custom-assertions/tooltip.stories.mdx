# Assertions

To help testing tooltip, new `assert` functions will be added. The simple way is to add the following code in `test-helper.ts`

```typescript
import * as QUnit from 'qunit';
import registerAssertions from '@upfluence/oss-components/test-support/register-assertions';
registerAssertions(QUnit.assert);
```

Now, it is possible de use `await assert.tooltip('element-class')` in your tests to easily check the different attributes
of a tooltip for a given element. This is the following list of tooltip custom assertion:

- [tooltip().exists()](#exists)
- [tooltip().doesNotExist()](#doesNotExist)
- [tooltip().hasTitle()](#hasTitle)
- [tooltip().hasSubtitle()](#hasSubtitle)
- [tooltip().hasIcon()](#hasIcon)
- [tooltip().doesNotHaveIcon()](#doesNotHaveIcon)
- [tooltip().doesNotHaveSubtitle()](#doesNotHaveSubtitle)
- [tooltip().hasPlacement()](#hasPlacement)
- [tooltip().isHtmlSafe()](#isHtmlSafe)
- [tooltip().isNotHtmlSafe()](#isNotHtmlSafe)

## <a name="exists"></a> assert.tooltip('element-class').exists()

`exists(trigger?: string, message?: string)`
