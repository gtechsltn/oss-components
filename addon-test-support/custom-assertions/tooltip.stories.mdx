# Testing

It's now possible to use `await assert.tooltip('element-class')` in your tests to easily check the different attributes
of a tooltip for a given element. Below is the list of the custom tooltip assertions:

- [tooltip().exists()](#exists)
- [tooltip().doesNotExist()](#doesNotExist)
- [tooltip().hasTitle()](#hasTitle)
- [tooltip().hasSubtitle()](#hasSubtitle)
- [tooltip().hasIcon()](#hasIcon)
- [tooltip().doesNotHaveIcon()](#doesNotHaveIcon)
- [tooltip().doesNotHaveSubtitle()](#doesNotHaveSubtitle)
- [tooltip().hasPlacement()](#hasPlacement)
- [tooltip().isHtmlSafe()](#isHtmlSafe)
- [tooltip().isNotHtmlSafe()](#isNotHtmlSafe)

## <a name="exists"></a> assert.tooltip('element-class').exists()

`exists(trigger?: string, message?: string)`

Checks the presence of the tooltip on a given element.

| Name         | Description |
|--------------| ------------|
|trigger (optional) | trigger name |
|message (optional) |	Short description of the actual expression |
